# Feature Priority — excer

> Stage 1.1 산출물. MVP 기능을 Must/Should/Could 로 정렬.

## 등급 정의
- 🔴 **MUST** — 빠지면 서비스가 성립 안 함. 1차 출시에 반드시 포함.
- 🟡 **SHOULD** — 빠지면 작동은 하지만 가치 크게 떨어짐. 출시 직후 ~ 첫 1~2주 안에 들어가야.
- 🟢 **COULD** — 있으면 좋음. 사용자 반응 보고 결정.

---

## 🔴 MUST (출시 필수)

### M1. 카카오맵 + 연습실 핀 + 클러스터링
- 서울 영역 표시, POI 핀, 줌 레벨에 따른 클러스터링
- 핀 클릭 → 상세 페이지로 이동
- **이유**: 이 기능이 서비스 그 자체.

### M2. 위치 검색 입력
- 지하철역 / 동 / "내 주변" 3가지 입력 방식
- 입력 → 지도 중심·줌 자동 이동
- **이유**: 발견의 첫 입구. 사용자는 항상 "어디 근처"부터 시작.

### M3. 연극 특화 필터 (거울 / 방음 / 평수)
- 거울: 있음 / 없음
- 방음: 등급 (강 / 중 / 약)
- 평수: 구간 (10평 이하 / 10~20평 / 20평 이상)
- **이유**: 일반 지도 대비 핵심 차별점. 이게 약하면 그냥 지도 쓰는 게 낫다.

### M4. 가격대 필터 + 시간당 가격 노출
- 카드/리스트에 "시간당 ₩○○○○" 표기
- 가격대 슬라이더 또는 구간 필터
- **이유**: 또 다른 차별점. 가격이 안 보이면 전화의 굴레로 돌아감.

### M5. 연습실 상세 페이지
- 상단: 사진 (최소 1장, 캐러셀)
- 중단: 시간당 가격, 운영시간, 위치, 연락처
- 정규화 속성 표 (거울/방음/평수/조명/대본대/마이크 — 있음·없음·등급)
- 하단 CTA: "전화하기" / "예약 페이지로 이동" (외부 링크)
- **이유**: 발견 → 행동(전화/예약 클릭) 전환 지점. 없으면 깔때기 끊김.

### M6. 비로그인 전체 조회
- 모든 발견·열람은 로그인 없이
- Clerk 는 연결만 해두고, 즐겨찾기·후기 등 후속 기능 슬롯만 마련
- **이유**: 마찰 0. 첫 방문에서 회원가입을 요구하면 이탈.

### M7. 시드 POI 50개 이상 (가드레일)
- 코드가 아니라 **데이터** 요구사항이지만 출시 전제
- 대학로/혜화/홍대/강남 중심으로 운영자가 수동 큐레이션
- **이유**: 빈 지도 출시 금지 — Stage 0 검증에서 합의.

---

## 🟡 SHOULD (출시 직후 1~2주 안에)

### S1. "마지막 확인일" 표기
- 상세 페이지·카드에 "○○일 전 확인" 노출
- **이유**: 데이터 신선도 신뢰 신호. 사용자가 데이터 정확도를 의심하기 시작하면 이탈.

### S2. 정렬 (가까운 순 / 가격 낮은 순)
- 리스트 뷰에서 동작
- **이유**: 초기 50개 핀일 땐 시각만으로도 됨. POI 100개+ 가 되면 필수.

### S3. 추가 연극 특화 필터 (조명 / 대본대 / 마이크)
- 거울·방음·평수 다음 우선순위
- **이유**: 시드 데이터 정규화가 끝나야 의미 있음. 단계적으로.

---

## 🟢 COULD (반응 보고 결정)

### C1. 검색어 자동완성
- 동·역 이름 입력 시 추천
- **이유**: 1차는 정적 리스트(서울 동·역)만으로 충분.

### C2. 지도 클릭 → 사이드 미리보기
- 핀 클릭 시 즉시 상세로 가는 대신 사이드 패널에 요약
- **이유**: 핀 → 상세 직행도 충분히 작동. UX 폴리시 단계 아이템.

---

- 실시간 빈 방 / 예약 / 결제
- 공급자 셀프 등록
- 서울 밖 지역
