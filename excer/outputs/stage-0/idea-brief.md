## 1. 한 줄 소개
서울 안 배우가 "지금 또는 곧 쓸 만한 연습실"을 지도에서 발견하고, **연극 관점 조건**(거울·조명·방음·평수·시간당 가격)으로 비교할 수 있는 웹 서비스.

## 2. 누구의 어떤 문제
- **타겟**: 20~30대 배우, 연극 전공생, 소극단 단원 (서울권 활동)
- **문제**: 일반 지도(네이버/카카오)에는 거울·조명·방음 같은 연극 특화 조건이 없고, 시간당 가격·실시간 빈 방 여부를 알려면 일일이 전화해야 한다. 동호회·소극단의 소규모 공간은 검색에 잡히지도 않는다.
- **현재 대체제**:
  1. 단톡방에 "연습실 추천ㅠㅠ" — 느리고 좁음
  2. 네이버/인스타 검색 — 정보 흩어지고 광고 노이즈
  3. 다니던 곳 계속 — 비교 불가능
- **핵심 통찰**: 정보의 부재가 아니라 **정리·비교의 부재**가 문제다.

## 3. 사용 상황
즉석(당일 오디션·연습)과 계획(정기 리딩 장소 결정) **둘 다** 강함.

## 4. 솔루션 콘셉트
일반 지도와의 차별점은 **두 가지뿐**:
1. 연극 특화 속성을 정규화해서 비교 가능
2. 시간당 가격을 일관된 형태로 노출

예약·결제는 MVP 에서 **하지 않음** (외부 링크/전화로 떨굼).

## 5. MVP 기능 목록 (`/plan` 의 입력)
1. **서울 지도 + 연습실 핀**
   - 카카오맵 SDK, POI 클러스터링
   - 지하철역/동/현재 위치 기반 검색
2. **필터 & 정렬**
   - 연극 특화 속성: 거울 / 방음 등급 / 평수 / 조명 / 대본대 / 마이크
   - 시간당 가격대
   - 정렬: 가까운 순 / 가격 낮은 순
3. **연습실 상세 페이지**
   - 사진, 시간당 가격, 운영시간, 위치, 연락처
   - 정규화된 속성 표 (있음/없음/등급)
   - "전화하기" / "예약 페이지로 이동" 외부 CTA
   - "마지막 확인일" 표기 (신선도 신호)
4. **비로그인 조회**
   - 모든 발견·열람은 로그인 없이
   - Clerk 연결만 해 두고 후속 기능(즐겨찾기/후기)용 슬롯 준비

## 6. 명시적 비-MVP (이번에 안 함)
- 사용자 후기/평점
- 사용자 제보 폼
- 즐겨찾기 / 최근 본 곳
- 실시간 빈 방·예약·결제
- 공급자 셀프 등록
- 서울 밖 지역

## 7. 데이터 전략
- **스코프**: 서울 전체. 시드는 대학로·혜화·홍대·강남 등 연극 밀집지부터.
- **시드 (수동)**: 운영자가 직접 큐레이션 **50개 이상**을 출시 전 확보 (빈 지도 출시 금지).
- **자동 수집**: 카카오 로컬 API 의 "키워드 장소 검색" 결과를 지도 표시용으로 호출. 자체 DB 에는 운영자가 검증·정규화한 데이터만 저장.
- **🚫 금지**: 스페이스클라우드 등 운영 사이트 스크래핑은 ToS 위반 위험으로 **하지 않음**. 필요시 공식 제휴 또는 공급자 직접 입력으로 전환.

## 8. 기술 스택 가설 (arch 패턴 차용)
- Next.js 16 App Router
- TypeScript, Tailwind v4, shadcn/ui
- DB: Neon Postgres + Drizzle ORM
- 인증: Clerk (acttub 공유 인스턴스)
- 지도: 카카오맵 JS SDK
- 호스팅: Vercel
- 패키지 매니저: **pnpm** (arch/comm 와 통일)
- basePath: `/excer` 하드코딩 (`next.config.ts`)

## 9. 디자인 톤

- 단톡방·acttub 채널 공유 5건 이상
- 주 활성 방문자 50명 이상
- "전화하기" / "외부 예약 페이지" CTA 클릭 발생
