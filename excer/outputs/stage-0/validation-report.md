> Stage 0.4 산출물. 데시러빌리티 / 피저빌리티 / 바이어빌리티 / 리스크 점검.

## 1. Desirability (사람들이 원하나?)
- ✅ **수요 신호 자체는 약하지 않음.** 사용자가 직접 "다들 연습실 정보를 편하게 얻고 싶음" 이라고 표현, 단톡방 추천 요청 패턴 = "정보 부재가 아니라 정리/비교 부재" 라는 가설을 지지.
- ⚠️ **유료화는 미지수.** MVP 는 무료 정보 도구. 수익화 가설은 아직 없음 — 일단 acttub 생태계 트래픽·리텐션 기여로 정당화.
- 🔍 **다음 검증 권장**: 시드 50개 데이터 + 랜딩 페이지 띄우고 단톡방 5~10곳에 던져 클릭/체류 측정. 6시간 정도 만들 수 있음.

### 데이터 수집 — 자세히
- **카카오/네이버 지도 API 자동 수집**: 카카오 로컬 API 의 "키워드로 장소 검색" 은 합법적 사용. 단, 결과를 자체 DB 에 영구 저장·재배포하는 것은 ToS 점검 필요. 안전한 패턴은 "지도 위에 표시할 때만 호출, 자체 DB 엔 자체 큐레이션 데이터만 저장".
- **스페이스클라우드 등 스크래핑**: 대부분의 운영자가 robots/ToS 로 금지. 합법 경로는 **공식 제휴 또는 운영자 직접 등록 요청**. 초기엔 운영자가 손으로 모으는 게 빠르고 안전.
- **시드 큐레이션 부담**: 50개 만들기 = 1~2일 작업. 200개 = 5~7일. 한 사람 사이드 프로젝트 기준 감당 가능.

## 3. Viability (지속 가능한가?)

- **운영 비용**: Vercel Hobby + Neon Free + Clerk Free + 카카오맵 무료 한도 → **월 0원** 으로 출발 가능.
- **수익 모델 옵션** (지금 결정 안 함, 메모만):
  - 연습실 공급자 광고/프로모션
  - 후기·평점 데이터 기반 프리미엄
  - acttub 통합 멤버십 일부
- **장기 해자**: 큐레이션 데이터 + 배우 관점 후기. 지도 API 만으로는 못 만드는 것.

## 4. Risks — 정직하게

| 리스크 | 영향 | 완화 |
| --- | --- | --- |
| **🔴 ToS 위반 스크래핑** | 법적·계정정지 | 자체 시드 + 카카오 API 정당 사용만, 스페이스클라우드 직접 수집은 금지 |
| **🔴 데이터 신선도** | 사용자 신뢰 즉시 붕괴 | 운영자 점검 주기(예: 월 1회) + "마지막 확인일" 노출 |
| **🟡 차별점 약함** | 네이버 지도로 충분 → 재방문 안 함 | 연극 특화 속성을 진짜 정규화·노출하는 게 핵심. 시드부터 이거 빠지면 의미 없음 |
| **🟡 cold start (POI 부족)** | 첫인상에서 "검색해도 없네" | MVP 출시 전 시드 50개 이상 확보 후 공개 |
| **🟡 운영 1인 부담** | 큐레이션·검수 누적 피로 | 6개월 안에 사용자 제보 + 검수 워크플로우 도입 |
| **🟢 카카오맵 호출 한도** | 트래픽 폭발 시 비용 | 일일 30만 호출 이내까지 무료, 초기 트래픽으론 한참 멀음 |

## 5. Go / No-Go 권고
**🟢 Go.** 단, MVP 정의에 다음 두 줄을 추가해야 함:

1. **시드 데이터 50개 이상**을 코드와 같이 준비. 빈 지도는 출시 금지.
2. **스크래핑 금지**. 카카오 API 는 호출용으로만 사용, 자체 DB 에는 운영자가 검증한 데이터만 들어감.
