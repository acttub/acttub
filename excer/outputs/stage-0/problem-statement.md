# Problem Statement — 연습실 찾기 (excer)

> Stage 0.2 산출물. 누구의 어떤 문제를, 왜 지금 풀어야 하는지.

## 한 문장 문제 정의
**배우/연극인은 자기 동선 안에서 "지금 또는 곧 쓸 만한 연습실"을 빠르게 찾고 비교하기 어렵다.**

## 페르소나
- **이름(대표)**: 20~30대 배우 / 연극 전공생 / 소극단 단원
- **상황**: 차 없음, 대중교통 이동. 평균 일주일에 1~3번 연습실 사용. 예산 민감.
- **기술 수준**: 스마트폰 네이티브. 굳이 별도 앱 깔기 싫어하는 편.

## 문제가 가장 아픈 순간 (Job-to-be-Done 트리거)
**두 가지 모두 강함:**

1. **즉석/당일** — 오디션 전날·당일, "지금 비는 곳" 급하게 잡기
2. **계획** — 제작처 정기 리딩·연습 장소를 몇 주 단위로 비교해서 결정

## 이 사람이 실제로 부딪히는 구체적 통점
| 통점 | 일반 지도가 못 풀어주는 이유 |
| --- | --- |
| 거울/조명/방음/평수 같은 연극 특화 조건이 안 보임 | 카테고리 자체가 일반 지도엔 없음 |
| 시간당 가격이 안 보임 | 가게 페이지에 안 적혀 있는 경우 많음, 일일이 전화해야 |
| 실시간 빈 방/예약 여부 모름 | 전화 안 하면 모름 |
| 동호회·소극단의 소규모 공간은 검색에도 안 나옴 | 사업자 등록·POI 등록 자체가 안 됨 |

## 현재 대체제 (사용자가 지금 "때우는" 방법)
- **단톡방에 추천 요청** ("연습실 추천ㅠㅠ") — 강력하지만 범위 좁고 느림
- **네이버/인스타에 "○○동 연습실" 검색** — 정보 흩어지고, 광고성 포스팅 노이즈
- **그냥 다니던 곳 계속 씀** — 한 번은 해결되지만 선택지·가격 비교 불가능

> 시사점: 사용자는 "발견(discovery)" 이 안 되어 단톡방·관성에 의존 중. 정보의 부재가 아니라 **정리·비교의 부재**가 핵심.

## 왜 지금인가
- 부모 프로젝트 `acttub` 가 배우 생태계(arch/comm/thea)를 이미 키우는 중 — 같은 사용자에게 자연스럽게 연결 가능
- 카카오/네이버 지도 API + 자체 큐레이션으로 MVP 가능 (자체 POI 빌딩 필요 없음)
- 예약·결제 같은 무거운 기능을 빼고 "정보 발견"에만 집중하면 진입 장벽이 낮음

## 비-목표 (이번엔 안 풀 문제)
- 실시간 예약/결제 통합 — 다음 단계
- 공급자(연습실 주인) 자체 가입 워크플로우 — 다음 단계
- 일반인(스터디·동호회 등) 대상 일반화 — 배우 집중이 차별점
