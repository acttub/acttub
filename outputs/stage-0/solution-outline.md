# Solution Outline — excer

> Stage 0.3 산출물. MVP 솔루션 윤곽.

## 한 줄 정의
**서울 안 배우가 "지금/곧 쓸 연습실"을 지도에서 발견하고, 연극 관점 조건으로 비교할 수 있는 웹 서비스.**

## 콘셉트
- 카카오/네이버 지도와의 차별점은 **두 가지뿐**: ① 연극 특화 속성(거울·조명·방음·평수)을 정규화해서 비교 가능, ② 시간당 가격 노출.
- 예약·결제는 **하지 않음** — 외부 링크 또는 전화 한 번으로 떨굼.
- 데이터는 운영자가 시드 + 외부 지도/연습실 관리 사이트(스페이스클라우드 류) 수집으로 채움.

## MVP 핵심 기능 (꼭 들어가는 것)
1. **서울 지도 + 연습실 핀**
   - 카카오맵 SDK 위에 POI 표시, 클러스터링
   - 현재 위치 / 지하철역 / 동 검색
2. **필터 & 정렬**
   - 연극 특화: 거울 여부 / 방음 등급 / 평수 / 조명 / 대본대 / 마이크
   - 가격대 (시간당)
   - 가까운 순 / 가격 낮은 순
3. **연습실 상세**
   - 사진, 시간당 가격, 운영시간, 위치, 연락처
   - 정규화된 속성 표 (있음/없음/등급)
   - "전화하기" / "예약 페이지로 이동" CTA (외부)
4. **로그인 없이도 조회 가능**
   - Clerk 는 연결만 해 두고, 즐겨찾기·후기 등 후속 기능을 위한 슬롯만 마련

## 차별점 (왜 일반 지도 대신 이걸 쓰나)
| 기준 | 일반 지도 | excer |
| --- | --- | --- |
| 거울/조명/방음 정보 | 없음 | 정규화 표 |
| 시간당 가격 비교 | 거의 없음 | 카드/리스트에서 바로 |
| 연극 동선(소극단 밀집지) 큐레이션 | 없음 | 시드 데이터로 채움 |
| 광고 노이즈 | 많음 | 없음 |

## 데이터 전략
- **시드(수동)**: 운영자가 서울 핵심 지역(대학로/홍대/강남/혜화 등) 50~200개 직접 큐레이션
- **자동 수집**: 카카오/네이버 지도 검색 API 로 "연습실" 후보 끌어오기 → 운영자 검수
- **연습실 관리 사이트 수집**: 스페이스클라우드 등에서 메타데이터 보강 (스크래핑은 ToS 확인 필수)
- **사용자 제보**: 다음 단계 (comm 와 연동)

## acttub 생태계 통합
- 도메인: `acttub.com/excer` (서브경로, 다른 앱과 동일 패턴)
- 인증: 부모 acttub Clerk 인스턴스 공유 (조회는 비로그인 허용)
- 디자인: `arch` 기준 (Pretendard, neutral surfaces, coral primary)
- 패키지 매니저: **pnpm** (arch/comm 와 통일, thea 와는 다름)

## 기술 스택 가설 (확정 아님)
- Next.js 16 App Router
- DB: Postgres (Neon) + Drizzle — arch 패턴 그대로
- 지도: 카카오맵 JS SDK
- 호스팅: Vercel (서브경로 rewrite 는 메인 acttub 도메인에서)
- basePath: `/excer` 하드코딩 (arch/comm 패턴)

## 비-MVP (다음 단계 후보)
- 즐겨찾기 / 최근 본 곳
- 배우 관점 후기·평점 (comm 연동)
- 사용자 제보 폼 + 운영자 검수
- 실시간 빈 방 (공급자 캘린더 연동)
- 예약/결제
- 공급자 셀프 등록 워크플로우
- 지방 확장
