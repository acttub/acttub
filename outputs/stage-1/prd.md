# PRD — excer (연습실 찾기)

> Product Requirements Document. Stage 0 (idea-brief) + Stage 1 (priority/stories/flow/screens) 통합본.
>
> 버전: v1 (MVP) · 작성일: 2026-05-20 · 상태: 디자인/엔지니어링 핸드오프 준비.

---

## 1. 개요

### 1.1 한 줄
**서울 안 배우가 "지금 또는 곧 쓸 만한 연습실"을 지도에서 발견하고, 연극 관점 조건(거울·방음·평수·시간당 가격)으로 비교할 수 있는 웹 서비스.**

### 1.2 작업 식별
- 코드네임: **excer**
- 위치: `C:\dev\acttub\excer` (그린필드)
- 라우팅: `acttub.com/excer` 서브경로 (arch/comm/thea 와 동일 패턴)
- 도메인 계열: acttub 배우/연극 생태계 4번째 앱

### 1.3 출시 목표 시점
- 시드 데이터 50개 확보 후 비공개 베타 → 일반 공개. (구체 일자 미정)

## 2. 문제 정의

### 2.1 페르소나
| 코드 | 누구 | 트리거 |
| --- | --- | --- |
| **U1** | 급한 배우 | 당일/내일 오디션 — 지금 비는 곳 |
| **U2** | 계획 배우 | 정기 연습 장소를 몇 주 단위로 결정 |

### 2.2 통점
1. 일반 지도에 거울·방음·평수 등 연극 특화 조건이 없다.
2. 시간당 가격이 보이지 않아 일일이 전화.
3. 소극단·동호회의 소규모 공간은 지도 검색에서 누락.
4. 결국 단톡방 추천에 의존 — 느리고 좁다.

**핵심 통찰**: 정보의 부재가 아니라 **정리·비교의 부재**가 문제다.

### 2.3 현재 대체제 & 한계
- 단톡방: 좁고 느림
- 네이버/인스타: 흩어지고 광고 노이즈
- 다니던 곳 관성: 비교 불가능

---

## 3. 목표 & 비-목표

### 3.1 목표 (MVP 가 달성해야 하는 것)
- G1. 서울에서 연극 특화 조건으로 연습실을 **3분 안에** 발견·비교할 수 있다.
- G2. 발견된 연습실에 대해 **전화 또는 외부 예약**으로 매끄럽게 전환된다.
- G3. 데이터는 적지만 **신뢰할 수 있다** (시드 50개+, 마지막 확인일 노출).
- G4. 진입 마찰 0 — **로그인 없이** 모든 발견·열람 가능.
- G5. acttub 생태계 안에서 자연스럽게 연결 (서브경로 + Clerk 공유).

### 3.2 비-목표 (이번엔 안 한다)
- 후기·평점
- 사용자 제보 폼
- 즐겨찾기 / 최근 본 곳
- 실시간 빈 방 / 예약 / 결제
- 공급자 셀프 등록
- 서울 밖 지역
- 네이티브 앱 (반응형 웹만)

### 3.3 성공 지표 (출시 후 4주 기준)
- 시드 POI **50개 이상** 등록, 80% 이상이 마지막 확인일 30일 이내
- 단톡방·acttub 채널 공유 5건 이상
- 주 활성 방문자 50명 이상
- 상세 진입 → 전화/외부 예약 CTA 클릭률 ≥ 20%
- 7일 재방문률 ≥ 15%

## 4. 기능 명세

### 4.1 MUST (출시 필수)

#### M1. 카카오맵 + 핀 + 클러스터링
- 카카오맵 JS SDK 위에 POI 핀 렌더
- 줌 레벨에 따라 핀 그룹핑 (cluster)
- 핀 클릭 → 상세 페이지로 이동
- 지도 영역 변경 (pan/zoom) → 리스트 자동 필터
- **수용기준**: 50개 POI 표시 시 첫 페인트 2초 이내 (NFR-01)

#### M2. 위치 검색
- 입력 방식 3가지: (a) 동 이름 (b) 지하철역 (c) "내 주변" 버튼
- 동/역 데이터는 정적 리스트 — MVP 자동완성은 COULD
- "내 주변": 브라우저 geolocation, 거부 시 서울 시청 좌표 폴백 (B1)
- **수용기준**: "혜화" 입력 → 지도 중심이 혜화로 이동, 500ms 이내

#### M3. 연극 특화 필터 (거울 / 방음 / 평수)
- 거울: 있음 / 없음
- 방음: 강 / 중 / 약 (등급)
- 평수: 10평 이하 / 10~20평 / 20평 이상
- 필터 적용 → 지도 핀 + 리스트 동시 갱신, URL 쿼리에 반영
- 결과 0건 → "조건을 풀어보세요" + 가장 가까운 완화 제안 (B3)

#### M4. 가격
- 카드/리스트에 "시간당 ₩○○○○" 표기
- 가격대 필터: 구간 (예: 1만 이하 / 1~2만 / 2만 이상) 또는 슬라이더 (디자인 결정)
- **수용기준**: 100% POI 에 가격 입력되어 있음. "협의" 같은 모호 값은 별도 표기.

#### M5. 연습실 상세 페이지 (`/rooms/[slug]`)
- 사진 캐러셀 (최소 1장 — 없으면 placeholder)
- 이름, 주소, 운영시간, 시간당 가격 (가장 강조), 연락처
- 정규화 속성 표: 거울 / 방음 / 평수 / 조명 / 대본대 / 마이크 (6항목)
- Sticky 하단 CTA:
  - "전화하기" → `tel:` 링크
  - "예약 페이지로 이동" → 외부 URL (없으면 숨김 — B6)
- 운영시간 외 진입 시 보조 텍스트 "현재 운영시간 아닐 수 있음" (B5)
- **수용기준**: 모바일에서 사진 → 가격 → CTA 까지 1스크롤 안에 도달

#### M6. 비로그인 조회
- 모든 발견·열람 라우트는 익명 접근
- Clerk 미들웨어 연결만 해두고 admin 라우트만 보호
- **수용기준**: 처음 진입한 사용자가 로그인 화면을 한 번도 만나지 않음

#### M7. 시드 POI ≥ 50개
- 출시 전 가드레일
- 대학로/혜화/홍대/강남 등 연극 밀집지부터
- **수용기준**: 시드가 50개 미만이면 출시 금지

### 4.2 SHOULD (출시 직후 ~ 1~2주 내)

#### S1. 마지막 확인일 표기
- 카드/상세에 "○○일 전 확인" 노출
- 30일+ 면 노란색 경고 배지 (B4)

#### S2. 정렬 (가까운 순 / 가격 낮은 순)
- 리스트 뷰에서 동작
- 기본값: 가까운 순

#### S3. 추가 필터 (조명 / 대본대 / 마이크)
- 시드 데이터의 정규화 충실도가 확인된 후 단계적으로 노출

### 4.3 COULD (반응 보고 결정)
- 검색어 자동완성 (동·역)
- 핀 클릭 → 상세 페이지 대신 사이드 미리보기

## 5. 사용자 흐름 요약

### 5.1 Flow A — 급한 배우
```
진입 → 지도 → ("내 주변" or 검색) → 핀 클릭 → 상세 → 전화하기
```
평균 30초 ~ 1분.

### 5.2 Flow B — 계획 배우
```
진입 → 지도 → 필터(거울/방음/평수/가격) → 리스트 비교 → 카드 클릭 → 상세 → 외부 예약 링크
```
평균 3~10분.

### 5.3 Flow C — 운영자 (시드/검수)
- MVP 1차: Drizzle Studio + Postgres 직접 입력
- 트래픽 100 DAU 또는 POI 100개 도달 시 어드민 UI 트랙 시작

## 7. 데이터 모델 (가설 — 엔지니어링이 확정)

```
Room
├─ id            (uuid)
├─ slug          (string, unique, kebab)         예: "hyehwa-acting-studio-1"
├─ name          (string)
├─ region        (string)                        예: "종로구 동숭동"
├─ subway        (string[])                      가까운 역 코드들
├─ lat / lng     (float)
├─ price_hour    (int, 원)
├─ price_note    (string, nullable)              "협의" 같은 모호값
├─ hours         (jsonb)                         { open: "09:00", close: "24:00", days: [...] }
├─ phone         (string)
├─ booking_url   (string, nullable)              외부 예약 페이지
├─ photos        (string[])                      URL 리스트 (Vercel Blob)
├─ mirror        (bool)
├─ soundproof    (enum: strong | medium | weak)
├─ size_pyeong   (int)
├─ lighting      (enum: bright | normal | dim | none)
├─ scriptstand   (bool)
├─ microphone    (bool)
├─ verified_at   (timestamp)                     마지막 확인일
├─ created_at / updated_at
└─ active        (bool, default true)
```

- DB: **Neon Postgres + Drizzle ORM** (arch 패턴 그대로)
- 사진: Vercel Blob (public)

## 11. 의존성 & 외부 환경

### 11.1 외부 서비스
- 카카오 디벨로퍼스 — JS 키 발급 필요 (출시 전)
- Neon — Postgres 인스턴스 (acttub 조직 안에 신규)
- Vercel — 프로젝트 신규 생성, 메인 acttub 도메인 rewrite 추가
- Clerk — 기존 acttub 인스턴스 재사용

### 11.2 운영 의존성
- 운영자 1인이 시드 50개 큐레이션 (예상 5~7일)
- 사진은 운영자가 직접 촬영 또는 공급자 허가받아 사용

## 12. 열린 질문 (다음 단계에서 결정)
