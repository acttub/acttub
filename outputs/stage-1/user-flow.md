# User Flow — excer

> Stage 1.3 산출물. 진입부터 전환(전화·예약 외부 이동)까지의 경로.

## 전제
- 비로그인 조회. 모든 흐름은 익명으로 시작.
- 주 진입 채널: acttub 생태계 내부 링크 / 단톡방·SNS 공유 링크 / 검색.
- 70% 이상 모바일 가정.

---

## Flow A — 급한 배우 (U1): "지금 비는 곳" 빠르게

```
[진입]                       [화면 1: 지도]
acttub.com/excer  ──►  서울 지도 + 핀 자동 표시
                            │
                            ├── A. "내 주변" 버튼
                            │      └─► 위치 권한 요청 ─► 현재 위치로 이동
                            │
                            └── B. 검색창에 "혜화" 입력
                                   └─► 지도 이동
                                            │
                                            ▼
                                     [핀 클릭]
                                            │
                                            ▼
                              [화면 2: 연습실 상세]
                              사진 / 가격 / 속성표 / 운영시간
                                            │
                            ┌───────────────┼───────────────┐
                            ▼                               ▼
                       [전환 1: 전화]                [이탈/뒤로가기]
                       tel: 링크 → 통화 앱             다른 핀 보러
```

**평균 소요**: 30초 ~ 1분. "지도 → 검색 → 핀 → 상세 → 전화" 4탭.

---

## Flow B — 계획 배우 (U2): 비교해서 정기 연습 장소 고르기

```
[진입] ──► [화면 1: 지도]
              │
              ▼
     [필터 패널 열기]
     ├─ 거울: 있음
     ├─ 방음: 강
     ├─ 평수: 10~20평
     └─ 가격: 시간당 15,000원 이하
              │
              ▼
     [지도 + 리스트 동시 갱신]
     필터 만족 핀만 표시 + 리스트에 카드 N개
              │
     ┌────────┴────────┐
     ▼                 ▼
[정렬 변경]         [카드 클릭]
가격 낮은 순             │
또는                     ▼
가까운 순           [화면 2: 상세]
     │                   │
     └──► 카드 클릭       │
                         ▼
              ┌──────────┴──────────┐
              ▼                     ▼
     [전환 2: 외부 예약]      [뒤로 가기]
     "예약 페이지로 이동"      목록에 표시되는
     → 외부 도메인            "방금 본 곳"
                            (브라우저 history)
```

**평균 소요**: 3~10분. 여러 카드 왕복 비교가 핵심.

---

## Flow C — 운영자 (시드/검수)

```
[관리 진입 — Clerk 로그인 + admin 권한]
            │
            ▼
    [관리 대시보드]
            │
   ┌────────┼────────┐
   ▼        ▼        ▼
[추가]   [수정]    [검수]
   │        │        │
   ▼        ▼        ▼
신규 POI  속성 갱신  "마지막 확인일"
정규화    가격 갱신   타임스탬프 갱신
저장      저장        (월 1회)
```

> MVP 에선 풀 어드민 UI 가 아니라 **Drizzle Studio + 시드 JSON** 으로도 시작 가능. 어드민 UI 는 SHOULD 가 아니라 운영 편의 기능으로 별도 트랙.

## 분기 포인트 정리

| # | 위치 | 분기 | 처리 |
| --- | --- | --- | --- |
| B1 | 첫 진입 | 위치 권한 거부 | 서울 시청 기본 좌표로 표시 |
| B2 | 검색창 | 매칭 결과 없음 | "근처 ○○로 이동하시겠어요?" 제안 |
| B3 | 필터 적용 | 결과 0개 | "조건을 풀어보세요" 안내 + 가장 가까운 필터 완화 제안 |
| B4 | 상세 진입 | 데이터 오래됨 (30일+) | "마지막 확인 ○일 전" 노란 배지 |
| B5 | 전화 CTA | 운영시간 외 | "현재 운영시간 아닐 수 있음" 보조 텍스트 |
| B6 | 예약 외부 링크 | 링크 없음 | CTA 숨김, 전화만 노출 |

## 핵심 전환 지표 (이 흐름이 잘 돌고 있는지 보는 신호)
- **진입 → 핀 클릭률** — 지도/검색 UX 가 작동하는지
- **상세 진입 → 전화 클릭률** — 정보가 결정으로 이어지는지
- **상세 진입 → 예약 외부 링크 클릭률** — 계획형 사용자 전환
- **재방문률 (7일/30일)** — 단톡방 대체제로 자리 잡는지

---

## 안 그리는 흐름 (이번 MVP 비대상 — 참고)
- 회원가입/로그인 (조회는 비로그인이라 불필요)
- 후기 작성·읽기
- 즐겨찾기 추가/리스트
- 사용자 제보
- 결제·예약 확정
