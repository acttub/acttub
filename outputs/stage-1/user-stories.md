# User Stories — excer

> Stage 1.2 산출물. 우선순위 잡힌 기능을 사용자 관점 문장으로.

## 페르소나
| 코드 | 누구 | 상황 |
| --- | --- | --- |
| **U1** | 급한 배우 | 당일·내일 오디션 대비, "지금 비는 곳" 빨리 찾기 |
| **U2** | 계획 배우 | 제작처 정기 연습 장소를 몇 주 단위로 고르는 중 |

## MUST 스토리

### M1. 지도 + 핀 + 클러스터링
- **US-01** (U1) — *나는* 사이트에 들어가자마자 *서울 지도와 연습실 핀이 보이길* 원한다, *그래야* 별도 클릭 없이 어디에 뭐가 있는지 한눈에 알 수 있다.
- **US-02** (U1) — *나는* 줌 아웃 했을 때 *핀이 동그라미 숫자로 묶여 보이길* 원한다, *그래야* 화면이 핀 폭탄으로 어지럽지 않다.
- **US-03** (U2) — *나는* 핀을 누르면 *그 연습실 상세 페이지로 바로 갈 수 있길* 원한다, *그래야* 자세한 정보를 한 번에 본다.

### M2. 위치 검색
- **US-04** (U1) — *나는* "혜화" 라고 치면 *지도가 혜화로 이동하길* 원한다, *그래야* 오디션 보러 가는 동선 안에서 찾는다.
- **US-05** (U1) — *나는* "내 주변" 버튼을 누르면 *현재 위치로 지도가 가길* 원한다, *그래야* 지금 있는 데서 가장 가까운 곳을 본다.
- **US-06** (U2) — *나는* "대학로역" 같은 지하철역 이름으로도 *검색할 수 있길* 원한다, *그래야* 차 없는 단원들 모이기 좋은 곳 기준으로 본다.

### M3. 연극 특화 필터 (거울/방음/평수)
- **US-07** (U1, U2) — *나는* "거울 있는 곳" 만 *필터해서 볼 수 있길* 원한다, *그래야* 동작 연습 가능한 곳만 추린다.
- **US-08** (U2) — *나는* "방음 강함" 으로 *필터해서 볼 수 있길* 원한다, *그래야* 발성 연습 가능한 곳만 본다.
- **US-09** (U2) — *나는* 평수 구간을 *고를 수 있길* 원한다, *그래야* 인원 수에 맞는 공간만 본다.

### M4. 가격
- **US-10** (U1, U2) — *나는* 카드/리스트에 *시간당 가격이 바로 보이길* 원한다, *그래야* 전화 안 하고도 예산 비교가 된다.
- **US-11** (U2) — *나는* "시간당 ₩○○○○ 이하" 로 *필터할 수 있길* 원한다, *그래야* 예산 안의 곳만 본다.

### M5. 상세 페이지
- **US-12** (U1, U2) — *나는* 상세 페이지에서 *사진을 여러 장 볼 수 있길* 원한다, *그래야* 가기 전에 분위기·상태를 파악한다.
- **US-13** (U1, U2) — *나는* 상세 페이지에서 *거울·방음·평수·조명·대본대·마이크가 표로 정리되길* 원한다, *그래야* 한눈에 조건을 비교한다.
- **US-14** (U1) — *나는* "전화하기" 버튼을 누르면 *바로 전화 앱이 열리길* 원한다, *그래야* 즉시 빈 방 확인 액션을 한다.
- **US-15** (U2) — *나는* "예약 페이지로 이동" 링크가 있으면 *그곳으로 바로 가길* 원한다, *그래야* 다시 검색 안 하고 예약한다.

### M6. 비로그인
- **US-16** (U1, U2) — *나는* 로그인 없이 *모든 발견·열람을 다 할 수 있길* 원한다, *그래야* 처음 들어와서 회원가입 막힘에 막혀 떠나지 않는다.

### M7. 시드 데이터 (가드레일 — 사용자 스토리는 운영자 관점)
- **US-17** (운영자) — *나는* 출시 전 *서울 핵심 지역에 최소 50개의 검증된 연습실 데이터를 등록할 수 있길* 원한다, *그래야* 첫 방문자가 "비어 있는 지도" 를 마주치지 않는다.

## SHOULD 스토리

### S1. 마지막 확인일
- **US-18** (U1, U2) — *나는* 각 연습실 카드/상세에 *"○○일 전 확인" 같은 신선도 표시가 있길* 원한다, *그래야* 정보의 정확도를 가늠한다.

### S2. 정렬
- **US-19** (U1) — *나는* 리스트를 *"가까운 순"* 으로 볼 수 있길 원한다, *그래야* 동선 우선으로 고른다.
- **US-20** (U2) — *나는* 리스트를 *"가격 낮은 순"* 으로 볼 수 있길 원한다, *그래야* 예산 우선으로 고른다.

### S3. 추가 필터
- **US-21** (U2) — *나는* 조명·대본대·마이크 같은 *세부 조건도 필터할 수 있길* 원한다, *그래야* 정기 연습 환경을 더 정밀하게 고른다.

---

## COULD 스토리

- **US-22** (U1) — *나는* 검색창에 글자를 치면 *동·역 추천이 뜨길* 원한다, *그래야* 빨리 입력한다.
- **US-23** (U1) — *나는* 핀을 누르면 *상세로 가기 전 사이드 미리보기로 핵심 정보(가격·평수·거울)를 보길* 원한다, *그래야* 여러 곳을 빠르게 훑는다.

---

## 비-기능 스토리 (인프라/품질)
- **NFR-01** — *나는* 첫 화면 진입 후 *2초 안에 지도가 보이길* 원한다 (Vercel + 카카오맵 SDK lazy load).
- **NFR-02** — *나는* 모바일에서도 *지도·필터·상세가 자연스럽게 동작하길* 원한다 (이용 환경 70%+ 모바일 가정).
- **NFR-03** — *나는* 연습실 데이터가 *틀린 정보로 노출되지 않길* 원한다 → 운영자 수동 검수 사이클(월 1회) 가드.
